## Phase D — code.json hygiene

### Findings from earlier today + this audit

- **8 GUID entries had stale `version` fields** — fixed in commit `729524d` (v3.1.0 sync)
- **PAGE_STANDINGS-000 was at v4 in registry while source was at v6** before today — pre-existing drift caught only because of the v3.1.0 work
- **LIB_VERSION-000/001 had drifted by 13 versions** (registry v18, source v30) — fixed today
- **4 more bumps in v3.1.5** — registry stayed in sync this time because GATE 4 caught new GUIDs and I manually bumped the existing ones

### Systematic risks

The version-drift problem is structural: the `/commit` GATE 4 catches **new** GUIDs added in the diff, but it does NOT detect when an existing GUID's source version-marker has been bumped without a registry update. That's why PAGE_STANDINGS-000 silently drifted by 2 versions and LIB_VERSION-000/001 by 13 each.

**Suggested fix:** GATE 4 should also `grep "// GUID:" -r app/src` and compare every GUID's source `vNN` against `code.json.version`. Any mismatch = registry update needed.

### Other code.json signals

- 1841 entries currently
- `total_guids` field stays in sync (verified by GATE 4 Check C)
- Broken-ref check (Check B) currently passes (verified at end of every commit)
