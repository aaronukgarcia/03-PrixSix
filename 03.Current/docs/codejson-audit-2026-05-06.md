## Remediation plan

### P0 — Naming consistency cleanup (1 hour)
The 52 broken refs are all caused by my choice today to register new entries with `-vNN` suffix in the `guid` field while older entries omit it. Two paths:

**Option A:** Rename the new entries to use bare ids (drop the `-vNN` suffix from the `guid` field, keep `version` field). This matches the older convention. ~15 entries from v3.1.0 + v3.1.5 + v3.1.7 affected.

**Option B:** Update the `dependencies` and `callChain` arrays in older entries to use the suffixed form for any reference to a new entry. More work, less consistent.

**Recommendation:** Option A. Renaming today's new entries is cleaner and matches the long-standing convention. One-shot script can do it.

Existing tooling. Batch-registers undocumented GUIDs found in source. Reduces undocumented count from 253 → 0 (or close to it). 5 minutes.
